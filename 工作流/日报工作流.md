# 日报工作流

## 一、格式规范

### 1.1 条目格式
`序号. 项目--子项目：完成内容；用时X小时；完成X%`

### 1.2 排版规则
- 所有条目**平级排列**，禁止缩进、分组、子标题、①②③符号
- 必须使用**阿拉伯数字序号**（1. 2. 3. ...）
- 今日完成与明日计划各自独立编号

### 1.3 输出规则
- 确认后用**两个代码块**分别输出，方便用户复制
- 代码块1：今日完成工作
- 代码块2：明日计划
- 每次修改后**立即重新输出完整两个代码块**，不要只说"改好了"

---

## 二、内容划分

### 2.1 项目-子项目映射（分类铁律）
用户说关键词时，按以下映射归类，**不要自作主张换项目**：

| 用户说的关键词 | 归到 | 示例 |
|---|---|---|
| 视觉识别、树莓派视觉方案、代码修改、工装、检测设备 | 智慧球阀--通用 | 视觉识别方案落地验证 |
| E3、易耘E3 | 施肥机--易耘E3 | E3样机供应链沟通 |
| E1、易耘E1 | 施肥机--易耘E1 | — |
| 软管泵、集成软管泵 | 施肥机--集成软管泵 | — |
| 双阀头 | 智慧球阀--双阀头 | — |
| 物料编码、培训、集团协助、杂项 | 其他 | 物料编码添加 |

### 2.2 完整项目列表
- 智慧运营中心项目
- 蓄收能力提升项目
- 智慧球阀（子项目：视觉识别/通用/双阀头等）
- 施肥机（子项目：集成软管泵/易耘E1/易耘E3等）
- 变频柜
- 其他

---

## 三、工时规则
- 工时合计**必须凑满8小时**
- 按用户说的时间分配填写，不要自行调整
- 工时不够8小时时，**主动问用户补哪项**
- 不要自己编项目凑工时

---

## 四、明日计划
- 根据今日**完成度<100%**的条目自动推导，可适当补充合理延续项
- **不带序号、不带项目前缀**，直接平述内容
- 不要瞎编，只基于今日未完成项+合理延续

---

## 五、流程
1. 日程提醒 → 问用户今天干了啥
2. 用户回复 → 按格式+映射归类整理
3. 输出两个代码块供用户确认
4. 用户修改 → 改完立即重新输出完整代码块
5. 确认无误 → 存入 `./日报/YYYY-MM-DD.md`

---

## 六、范例

```
3. 施肥机--易耘E3：E3样机供应链沟通；用时1小时；完成50%
4. 其他：物料编码流程修正；用时1小时；完成100%
5. 其他：物料编码添加；用时1小时；完成90%
6. 其他：慧丰收小程序培训；用时1小时；完成100%
```

```
继续视觉识别方案落地验证及试用

```

## 七、明日计划参考模板
用户常见的明日计划方向，可在推导时参考：
- 视觉方案定版启用测试
- E3和直通阀进度跟踪
- 软著相关集团协助
- 软管泵施肥机相关研发任务
- 集成软管泵PRD与方案完成


---

## 八、未确认续催规则
- 用户没有明确回复"已提交"或类似确认语 → **继续催促**，不要停止
- 只有用户明确说"已提交"或确认完成 → 才停止催促
- 避免：用户沉默/忽略 → 误以为已完成 → 停止催
