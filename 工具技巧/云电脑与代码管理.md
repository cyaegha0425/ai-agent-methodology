# 云电脑与代码管理参考

> 详细内容见 `./文档工作/AI协作驱动开发方法论 v1.0.md` 第六章

## 云电脑能力边界

### 能做
- bash执行任何终端命令
- 读写文件、修改代码
- 依赖持久保存
- agent-browser控制浏览器访问网站

### 不能做
- 访问部分海外网站（国内网络环境）
- 操作用户本地环境（完全隔离）
- 看到用户屏幕（无GUI）

### 来源
1. 扣子高阶订阅：开箱即用
2. OpenClaw自部署：独立主机或云服务器，效果一样

## OpenClaw部署要点

### 三种方式
1. 云服务器一键部署（腾讯云Lighthouse/DigitalOcean预装镜像）
2. Docker部署（额外隔离层）
3. 独立本地机器安装（旧笔记本/NUC/树莓派）

### 核心原则
- **不要用主力电脑部署**——Agent有执行权限，隔离是安全边界也是优点
- 部署前需准备LLM API Key
- 本地部署优势：零月租、数据在手、Agent无法触碰本机
- 云服务器优势：7×24在线
- 必须配置渠道白名单

### Docker快速部署
```bash
docker pull openclaw/openclaw:latest
docker run -d --name openclaw -p 8080:8080 \
  -e ANTHROPIC_API_KEY=YOUR_API_KEY \
  -v /opt/openclaw/data:/app/data \
  --restart unless-stopped openclaw/openclaw:latest
```

### 本地安装
```bash
npm install -g openclaw@latest
openclaw onboard
openclaw gateway start
```

## Git代码管理场景

### 用户不需要记命令
描述意图→Agent选命令。常见场景：
- 推代码：`git add → commit → push`
- 建仓库：GitHub API创建→clone→初始化
- 改仓库名：GitHub API修改
- 看记录：`git log`整理返回
- 回退版本：`git reset`/`git revert`

### 多分支管理
- 创建分支：`checkout -b → push`
- 切换修改：`checkout 分支 → 修改 → commit → push`
- 合并分支：`checkout main → merge → push`

### Force Push后同步（踩过的坑）
用户本地必须执行（不能git pull，会报错历史不一致）：
```powershell
git fetch origin
git reset --hard origin/分支名
```

## 远程调试工作流

```
用户本地发现问题 → 截图/报错给Agent → Agent分析修改 → push
→ 用户fetch+reset同步 → 测试 → 反馈结果
```

### 关键场景
- 打包后缺模块：PyInstaller遗漏懒加载→加--hidden-import
- 改代码不生效：编译缓存(.pyd)优先级高于源码→开发阶段删.pyd
- 多文件关联修改：Agent全局grep一次性改完

## 环境隔离

```
用户电脑 ←→ GitHub ←→ Agent云电脑
运行/测试      代码同步     修改/分析
截图/日志                  文件生成
```

- 代码通过GitHub同步（唯一可靠桥梁）
- 文件通过对话传递
- Agent无法看到用户屏幕
- GUI问题请截图，报错请复制文字

## GitHub仓库操作：API优先
- 建仓库不要用浏览器/agent-browser，直接curl调GitHub API更快更稳
- `gh auth login` 可能因scope不足失败，但API token可以直接用curl操作
- 示例：`curl -X POST https://api.github.com/user/repos -H "Authorization: token <TOKEN>" -d '{"name":"repo-name","public":true}'`
- push时如果remote用https+token内嵌方式，注意格式：`https://<user>:<token>@github.com/<user>/<repo>.git`
- 默认分支可能是master而非main，push前先`git branch -a`确认
