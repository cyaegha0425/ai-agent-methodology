# 知识库使用方法

## 机制
- INDEX.md 是总索引，记录所有分类和文件
- 遇到相关问题时，先读INDEX.md定位分类文件夹
- 再按需读取对应文件，不全量加载
- 目的：节省token和积分消耗

## 分类
- RSI星际公民 — DOM结构、开发笔记、API参考
- 编程开发 — Playwright技巧、Python打包、tkinter笔记
- 扣子平台 — 能力边界、积分使用
- 邮件系统 — 分拣规则、邮箱配置
- 工具技巧 — sub-agent规范等
- Agent运维 — 信用点追踪、邮件分身管理
- 安全 — 安全原则
- 其他 — 待补充

## 维护
- 新知识及时写入对应分类文件夹
- 更新INDEX.md索引
- 保持文件简短精炼，方便按需读取

## 两层INDEX体系

### 全局INDEX（`./知识库/INDEX.md`）
- 管理所有分类目录和项目入口
- 每个分类下列出文件清单和一句话摘要
- 每个项目入口指向项目INDEX

### 项目INDEX（项目目录内的INDEX.md）
- 管理项目内所有文件、关键决策、当前状态
- 项目相关知识归到项目INDEX下管理
- 全局INDEX只引用项目INDEX，不重复列出项目内文件

### 判断标准
- 项目专属知识 → 项目INDEX管（如RSI的API、DOM结构、harness）
- 跨项目通用知识 → 全局INDEX分类管（如Playwright通用技巧、Python打包通用方法）

## 知识库清理机制

### 淘汰标准
以下内容应从知识库清理或合并：
- 已完成项目的开发笔记，且不再需要参考的 → 压缩为"经验总结"段落，删除详细过程
- 同一信息出现在多个文件中 → 合并到一处，其他文件只留引用
- 超过3个月未访问且不属于长期参考的技术文档 → 评估是否需要保留

### 清理频率
- 每月检查一次知识库总大小，超过80KB时触发清理
- 项目结项时做一次归档清理
