# TRAE SOLO 速通指南

> 写给阿离：产品经理的 AI 搭子，专门搞定「写东西+调代码」那些烦心事

---

## 这玩意儿能干啥？

TRAE SOLO 就像你的 **AI 办公室助理**，有两个模式：

**MTC 模式** = 你的文秘
- 帮客户做 PPT
- 生成操作手册 Word
- 洗数据、做报表

**Code 模式** = 你的程序员小弟
- 在树莓派上写 Python 代码
- 调 OpenMV / OpenCV 程序
- 查 bug、修代码

---

## 一、先装上！10分钟搞定

### 方式1：网页版（推荐先试试）

打开 [solo.trae.cn](https://solo.trae.cn)，用手机号登录就行，0 门槛。

优点：不用安装，手机都能用
缺点：临时用用还行，复杂项目建议用桌面端

### 方式2：Windows 桌面端（长期用这个）

1. 打开 [trae.cn](https://trae.cn)，点「下载 SOLO 桌面端」
2. 下载 Windows 安装包，双击安装
3. 打开后用手机号登录
4. 搞定！

装好后长这样：
- 左边：任务列表
- 中间：跟 AI 聊天的地方
- 右边：文件/预览区

## 二、两个模式怎么选？

看任务类型：

| 任务 | 用哪个模式 |
|------|-----------|
| 做 PPT 给客户看 | MTC |
| 写操作手册 Word | MTC |
| 洗数据出报表 | MTC |
| 写 Python 代码 | Code |
| 调树莓派程序 | Code |
| 查报错原因 | Code |

**切换方法**：点左上角那个模式切换按钮，MTC ↔ Code 随时切

## 三、MTC 模式实操

### 场景1：5分钟生成客户汇报 PPT

**直接跟 AI 说：**
> 帮我做一份球阀检测系统的客户汇报 PPT，包含：
> - 项目背景（OpenMV 到双摄的升级）
> - 技术方案
> - 预期效果
> - 交付时间

AI 会自动生成完整 PPT，你只需要：
1. 检查内容对不对
2. 换个好看的配色
3. 下载导出

### 场景2：生成操作手册 Word

**跟 AI 说：**
> 根据这个功能需求文档，生成一份操作手册：
> - 包含硬件接线说明
> - 软件安装步骤
> - 常见问题 FAQ
> - 输出格式：Markdown，方便转 Word

### 场景3：洗数据出报告

把 CSV 文件丢给 AI，说：
> 这是一份检测数据，帮我：
> - 清洗异常值
> - 按日期统计通过率
> - 生成折线图
> - 输出一份分析报告

代码调好后，通过 Git 推送到仓库，然后在树莓派上：

## 五、和阿音的协作流程

> 核心思路：阿音出设计 → TRAE SOLO 写代码/出文档 → 交接靠 GitHub + Markdown

### 流程图

```
阿音（扣子AI）
    ↓ 输出一份设计文档
    ↓
TRAE SOLO（MTC模式）
    ↓ 接收文档，生成 PPT/操作手册
    ↓
TRAE SOLO（Code模式）
    ↓ 接收代码需求，写代码
    ↓ GitHub 推送
    ↓
树莓派（git pull 拉取最新代码）
```

### 详细步骤

**Step 1：阿音出设计文档**

让阿音生成一份 Markdown 文档，保存到 GitHub 或者直接复制给你。

**Step 2：TRAE SOLO 读取文档**

在 TRAE 里把文档丢给 AI：
> 根据这份需求文档，帮我：
> 1. 生成客户汇报 PPT
> 2. 生成操作手册 Word

**Step 3：TRAE SOLO 写代码**

如果阿音也给了代码需求，直接在 Code 模式里：
> 请根据这个需求实现双摄同步采集功能，代码要求：
> - Python 3
> - 用 OpenCV
> - 支持同时读取两个摄像头

**Step 4：GitHub 交接**

代码写完后，在 TRAE 里：
1. 点「源代码管理」
2. 写个 commit 信息（比如「feat: 双摄同步采集 v1.0」）
3. 点「推送到 GitHub」

**Step 5：树莓派拉取**

在树莓派终端：
```bash
cd ~/ball-valve-detect
git pull origin main
python main.py
```

## 六、高频问题

**Q：MTC 和 Code 模式数据互通吗？**
A：同一个项目里可以切换，AI 会记得上下文。

**Q：代码写完了怎么验证？**
A：Code 模式自带终端和预览功能，可以直接跑 Python 代码看效果。

**Q：树莓派上能直接用 TRAE SOLO 吗？**
A：不行，树莓派性能不够。建议在 Windows 上写代码，通过 Git 部署到树莓派。

**Q：GitHub 连接不上怎么办？**
A：检查 SSH 密钥配置，或者用 HTTPS 方式克隆仓库。
