# 双轨审查工作流

## 适用场景
🔴 高风险任务必须走此流程：
- 批量删除文件
- 覆盖核心配置文件
- 跨项目迁移
- 涉及敏感信息的操作

## 三条硬约束
1. **主agent出方案，sub-agent执行，交叉验收** — 防止自我服务
2. **高风险操作必须有回滚方案** — 不能只管杀不管埋
3. **执行前必须用户确认** — 没有例外

## 流程

### 阶段1：主agent起草
```
1. 识别高风险任务
2. 编写详细方案（含风险评估）
3. 准备回滚方案
4. 输出给用户确认
```

### 阶段2：用户确认
```
方案确认：✓ / ✗
风险知情：✓ / ✗
授权执行：✓ / ✗
```

### 阶段3：sub-agent执行
```
- 按方案执行
- 记录执行日志
- 出现异常立即停
```

### 阶段4：交叉验收
```
主agent验收产出
↓
引用给用户看
↓
用户最终确认
↓
完成/回滚
```

## 评分维度

| 维度 | 说明 |
|------|------|
| 完整性 | 需求是否全部覆盖 |
| 准确性 | 产出是否符合意图 |
| 安全性 | 有无副作用/风险 |
| 可回滚性 | 出问题能否恢复 |

## 修订规则
- 主agent审查发现不合格 → 退回sub-agent重做
- 用户发现不合格 → 指出问题 → 主agent修订方案 → 再执行
