# 跨渠道提醒配置

## 典型场景
- 吃药提醒
- 打卡提醒（运动、读书等）
- 喝水提醒
- 定时复查提醒

## 核心问题
跨渠道任务的关键挑战：**用户在微信/其他渠道回复确认，但Agent不知道**

## 常见问题
| 问题 | 后果 |
|------|------|
| 只发提醒，不约定确认机制 | 不知道用户是否完成 |
| 用户确认后不回写 | 重复提醒或遗漏 |
| 依赖Agent"记住" | Agent重启后忘记 |

## 正确做法

### 第一步：创建提醒时约定确认机制
```
Calendar提醒：
- 时间：XX:XX
- 内容：提醒吃药
- 确认方式：用户回复"已吃药"
- 回写位置：知识库/打卡记录.md
```

### 第二步：配置捕获机制
```
邮件分身/其他渠道捕获：
- 关键词过滤："已吃药"、"吃药了"
- 提取时间+状态
- 自动回写到知识库
```

### 第三步：确认后必须回写
```
捕获到确认 → 提取关键信息 → 写入记录文件
记录内容：
## 吃药记录
| 日期 | 时间 | 药品 | 确认方式 |
|------|------|------|---------|
| 2026-01-01 | 08:00 | 维生素 | 微信回复 |
```

### 第四步：下次提醒时检查记录
```
提醒逻辑：
1. 查打卡记录.md
2. 今天已确认？→ 不提醒/轻提醒
3. 今天未确认？→ 正常提醒
```

## 禁止做法
- ❌ 只发提醒，不约定确认机制
- ❌ 用户确认后不写记录
- ❌ 依赖Agent"记住"而不写下来
- ❌ 重复提醒已确认的任务

## 配置示例

### 吃药提醒Calendar
```yaml
summary: "提醒吃药"
description: |
  ## 任务
  提醒用户吃维生素D
  
  ## 确认方式
  用户微信回复"已吃药"
  
  ## 回写位置
  ./知识库/打卡记录.md
  
  ## 注意
  执行后检查打卡记录，今日已确认则轻提醒
time: "2026-01-01T08:00:00+08:00"
rrule:
  freq: "daily"
  interval: 1
```

```markdown
| | | | |

## 读书记录
| 日期 | 书名 | 页数 | 状态 |
|------|------|------|------|
| | | | |
```
