# 知识库使用方法

## 机制
- INDEX.md 是总索引，记录所有分类和文件
- 遇到相关问题时，先读INDEX.md定位分类文件夹
- 再按需读取对应文件，不全量加载
- 目的：节省token和积分消耗

## 分类
- 工作流 — 标准操作流程（日报、审查、场景工作流）
- Agent运维 — 平台配置与维护
- 工具技巧 — 工具使用规范

## 维护
- 新知识及时写入对应分类文件夹
- 更新INDEX.md索引
- 保持文件简短精炼，方便按需读取

## 两层INDEX体系

### 全局INDEX（`./知识库/INDEX.md`）
- 管理所有分类目录和项目入口
- 每个分类下列出文件清单和一句话摘要

### 分类INDEX（分类目录内的INDEX.md）
- 管理该分类下所有文件
- 记录关键决策、当前状态

### 判断标准
- 项目/任务专属知识 → 分类INDEX管
- 通用知识 → 全局INDEX分类管

## 知识库清理机制

### 淘汰标准
以下内容应从知识库清理或合并：
- 已完成任务的详细过程 → 压缩为"经验总结"段落
- 同一信息出现在多个文件中 → 合并到一处
- 超过3个月未访问且不属于长期参考 → 评估是否需要保留

### 清理频率
- 每月检查一次知识库总大小，超过80KB时触发清理
- 任务结项时做一次归档清理

### 快速检查脚本
```bash
# 列出所有.md文件
find 知识库 -name "*.md" -type f

# 检查哪些文件没被INDEX引用
for f in $(find 知识库 -name "*.md"); do
  if ! grep -q "$f" 知识库/INDEX.md; then
    echo "❌ 未被INDEX引用: $f"
  fi
done
```
