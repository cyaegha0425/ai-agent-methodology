# 文档处理技巧

> 从 TOOLS.md 迁出的技术实现细节，用时按需查阅

## docx文档生成
- 中文字符串中包含双引号时，需用单引号包裹整个字符串（如`text: '研发面向规模化、高标准农田的"固液两用智能施肥机"'`），避免JavaScript语法错误
- 输出文件路径需注意执行脚本的当前目录，避免相对路径错误（如脚本在`./文档工作/`下，直接写`'慧飞固液两用施肥机-结项报告V1.0.docx'`而非`'./文档工作/...'`）
- 依赖安装：生成docx需先执行`npm install docx`安装依赖
- 验证方法：可使用`pandoc "文档名.docx" -o 验证文件.md`转换为markdown验证内容完整性

## PDF转图片
- 使用PyMuPDF(fitz)库进行PDF转图片，安装命令：`pip install PyMuPDF`
- 设置zoom=2.0可以达到150 DPI的分辨率，满足文档需求
- 保存图片时使用`page_1.png`、`page_2.png`等命名规则，便于后续引用
- 输出目录需提前创建，使用`mkdir -p ./目录名`确保目录存在

## 文档修改与图片插入
- 先确认文档中需要插入图片的位置格式，包括换行符、引用符号等细节
- 对于markdown文档，图片引用格式：`![模块名原理图](./图片路径/page_X.png)`
- 修改文档时，优先使用精确字符串替换而非正则表达式，避免格式匹配问题
- 替换前先查看目标位置的原始格式，确保替换字符串与原格式完全一致
- 批量修改时，建议先测试单个替换，确认成功后再进行批量操作
