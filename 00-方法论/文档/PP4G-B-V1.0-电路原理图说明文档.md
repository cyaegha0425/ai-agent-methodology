## 📖 第一章：这块板子是干嘛的？

**PP4G-B** 是一块专门为**泵房监控系统**设计的**嵌入式控制板**。你可以把它理解为一个"泵房数据采集与远程传输的智能终端"。

### 它的核心工作

想象一下，泵房里有很多设备需要监控：
- 水池里还有多少水（**液位**）
- 水管里的水压够不够（**压力**）
- 水流得快不快（**流量**）
- 电池还能用多久（**电池电压**）

PP4G-B 板子的工作就是：**采集这些数据 → 处理分析 → 通过4G网络上传到云端服务器**。这样，运维人员坐在办公室里就能实时看到泵房的运行状态，不用天天跑现场。

### 典型应用场景

| 场景 | 描述 |
|------|------|
| 市政供水泵房 | 监控水厂、污水泵站的运行状态 |
| 楼宇供水系统 | 监测二次供水设备的液位、压力 |
| 农业灌溉泵站 | 远程监控农田灌溉系统 |
| 工业循环水系统 | 监控化工厂、制药厂的循环水设备 |

### 3.1 MCU模块 —— 板子的"大脑"

#### 一句话总结
> **MCU是整个板子的指挥中心**，负责接收各路信号、处理数据、协调各个模块工作，就像工厂里的总调度室。

#### 关键芯片

| 芯片型号 | PY32F030K28U6TR |
|----------|------------------|
| **类型** | 32位ARM Cortex-M0+ 单片机 |
| **主频** | 最高48MHz（相当于48百万次/秒的运算速度） |
| **Flash** | 32KB（相当于能存3万多个汉字的程序空间） |
| **SRAM** | 4KB（相当于临时工作台，掉电数据会丢失） |
| **工作电压** | 1.8V ~ 3.6V |
| **封装** | TSSOP20（20个引脚的小芯片） |

#### 工作原理（产品经理能懂的语言）

**你可以这样理解MCU：**

MCU就像一个**超级能干的秘书**：
- 它有很强的**记忆力**（Flash存程序）
- 有个**临时记事本**（SRAM处理数据）
- 有很多**不同功能的电话线**（GPIO引脚可以配置成不同用途）
- 能以**极快的速度**处理各种事情（48MHz主频）

**MCU具体做什么？**
1. **接收数据**：从ADC读取传感器数据，从4G模块接收指令
2. **处理数据**：把原始电压值换算成实际的物理量（如"12.5米液位"）
3. **发送指令**：控制4G模块发送数据、控制继电器开关
4. **协调工作**：确保各个模块按正确的顺序工作

#### 关键信号/引脚

| 引脚名 | 方向 | 功能说明 |
|--------|------|----------|
| PA0~PA7 | 输入/输出 | **ADC通道**，连接各种传感器（液位、流量、压力等） |
| PB6/PB7 | 输入/输出 | **I2C总线**，用于和EEPROM、蓝牙模块通信 |
| PA9/PA10 | 输入/输出 | **USART串口**，用于GPS模块通信 |
| PB3~PB5 | 输入/输出 | **SPI总线**，用于读取外部Flash |
| PA13/PA14 | 输入/输出 | **SWD调试接口**，烧录程序、在线调试 |
| NRST | 输入 | **复位引脚**，让MCU重新开始工作 |

### 3.2 4G通信模块 —— 板子的"嘴巴和耳朵"

#### 一句话总结
> **4G模块是板子的通信枢纽**，通过4G网络把采集到的数据发送到云端，同时接收云端下发的指令。

#### 关键芯片

| 芯片型号 | EC800KCNLC |
|----------|-------------|
| **品牌** | 移远通信（Quectel） |
| **类型** | LTE Cat.1 无线通信模块 |
| **网络制式** | 4G LTE（支持移动/联通/电信） |
| **下行速率** | 最大10Mbps |
| **上行速率** | 最大5Mbps |
| **工作电压** | 3.4V ~ 4.3V（典型3.8V） |
| **尺寸** | 17.7mm × 15.8mm × 2.4mm |
| **SIM卡** | 支持1.8V和3.0V SIM卡 |

#### 工作原理（产品经理能懂的语言）

**你可以这样理解4G模块：**

EC800K就像一个**专业的信使**：
- 它内置了完整的**4G modem**（相当于手机里的通信芯片）
- 它有自己的**处理器和内存**，可以独立运行TCP/IP、MQTT等网络协议
- 它通过**AT指令**和MCU对话（MCU告诉它"发送这条数据"，它就执行）

**和MCU的通信方式：**
MCU和4G模块之间通过**串口（UART）**进行通信，有点像两个人通过打字聊天：
- MCU发送AT指令：`AT+QIOPEN=1,0,"TCP","cloud.example.com",1883`
- 4G模块回复：`+QIOPEN: 1,0,0`（表示连接成功）

#### 关键信号/引脚

| 信号名 | 方向 | 功能说明 |
|--------|------|----------|
| MOD_LOG_TXD | MCU→4G | **日志串口发送**，调试用 |
| MOD_LOG_RXD | 4G→MCU | **日志串口接收**，调试用 |
| MOD_RESET | MCU→4G | **复位信号**，MCU控制模块重启 |
| MOD_BOOT | 输入 | **启动模式**，接测试点 |
| MOD_CH_CHRG | 输出 | **充电状态**，指示正在充电 |
| MOD_CH_DONE | 输出 | **充电完成**，指示充电结束 |
| MOD_MCU_INT | 输出 | **中断信号**，通知MCU有事件 |
| MOD_BT_WKP | 输入 | **蓝牙唤醒**，蓝牙可唤醒4G模块 |
| MOD_MCU_WKP | 输出 | **MCU唤醒**，4G可唤醒MCU |

### 3.3 电平转换模块 —— 板子的"电压翻译官"

#### 一句话总结
> **电平转换芯片解决了一个关键问题**：板子上不同芯片的工作电压不同，需要一个翻译让它们能正常交流。

#### 关键芯片

| 芯片型号 | RS0108YQ20 |
|----------|-------------|
| **品牌** | 润石科技（RUNIC） |
| **类型** | 8位双向电平转换器 |
| **通道数** | 8通道 |
| **数据速率** | 最高24Mbps |
| **VCCA（低压侧）** | 1.65V ~ 5.5V |
| **VCCB（高压侧）** | 2.3V ~ 5.5V |
| **封装** | TSSOP20 |

#### 工作原理（产品经理能懂的语言）

**问题背景：**

在这块板子上，不同芯片的工作电压不同：
- **MCU (PY32F030)**：工作电压 1.8V
- **4G模块 (EC800K)**：工作电压 3.0V
- **蓝牙模块**：工作电压 3.0V

如果直接把1.8V的信号接到3.0V的芯片上，就像用**手机充电线直接插到工业机器上**——可能完全没反应，甚至可能损坏芯片！

**RS0108的作用：**

这个芯片就像一个**专业的翻译**，能把：
- 1.8V信号 ↔ 3.0V信号
- 3.3V信号 ↔ 5V信号

它支持**双向转换**，信号可以往两个方向传输，而且**自动识别方向**，不需要软件配置。

#### 转换关系速查

| 本板应用 | 转换方向 | 用途 |
|----------|----------|------|
| 蓝牙串口 | 1.8V ↔ 3.0V | MCU和蓝牙模块的串口通信 |
| I2C总线 | 1.8V ↔ 3.0V | MCU和EEPROM的I2C通信 |

### 3.4 SPI Flash模块 —— 板子的"仓库"

#### 一句话总结
> **SPI Flash是大容量存储芯片**，用于存储程序固件和需要长期保存的数据，即使断电也不会丢失。

#### 关键芯片

| 芯片型号 | ZD25WD20BEI |
|----------|--------------|
| **容量** | 2Mbit（256KB） |
| **接口** | SPI（串行外设接口） |
| **工作电压** | 1.65V ~ 3.6V |
| **数据速率** | 最高104MHz |
| **擦写寿命** | 10万次 |
| **数据保持** | 20年 |
| **封装** | SOP8 |

#### 工作原理（产品经理能懂的语言）

**Flash和普通内存的区别：**

- **SRAM（MCU内部）**：像**草稿纸**，速度快，但断电后内容全丢
- **Flash**：像**印刷的书**，速度稍慢，但**断电后内容永久保存**

**这块Flash的用途：**

| 用途 | 说明 |
|------|------|
| **固件存储** | MCU的程序代码通常存放在这里 |
| **日志存储** | 采集到的数据可以临时存在这里，等网络好了再上传 |
| **配置备份** | 存储设备的配置参数 |

#### 关键信号/引脚

| 信号名 | 方向 | 功能说明 |
|--------|------|----------|
| FLASH_CS | MCU→Flash | **片选信号**，选通这个Flash芯片 |
| SPI1_MISO | Flash→MCU | **数据输出**，Flash把数据发给MCU |
| SPI1_MOSI | MCU→Flash | **数据输入**，MCU把数据写入Flash |
| SPI1_SCK | MCU→Flash | **时钟信号**，同步数据传输 |

### 3.5 EEPROM模块 —— 板子的"记事本"

#### 一句话总结
> **EEPROM是小容量但高可靠性的存储芯片**，专门用来存储需要频繁修改、但又必须断电保存的关键参数。

#### 关键芯片

| 芯片型号 | BL24C256A |
|----------|------------|
| **容量** | 256Kbit（32KB） |
| **接口** | I2C（两线串行接口） |
| **工作电压** | 1.8V ~ 5.5V |
| **通信速率** | 最高1MHz |
| **擦写寿命** | 100万次 |
| **数据保持** | 100年 |
| **封装** | SOP8/TSSOP8 |

#### 工作原理（产品经理能懂的语言）

**EEPROM vs Flash：**

| 特性 | EEPROM | SPI Flash |
|------|--------|-----------|
| **容量** | 较小（32KB） | 较大（256KB~更大） |
| **速度** | 较慢（写入需3-5ms） | 较快 |
| **擦写粒度** | 可按**字节**修改 | 需按**扇区**擦除（4KB起） |
| **寿命** | 100万次 | 10万次 |
| **适用场景** | 配置参数、日志 | 固件、大量数据 |

**通俗理解：**
- **Flash**像一本**装订好的书**，想改内容得撕掉整页重来
- **EEPROM**像**白板**，想改直接擦掉重写

**本板EEPROM存储内容：**

| 内容 | 说明 |
|------|------|
| 设备ID | 设备唯一身份标识 |
| 校准参数 | 传感器校准系数 |
| 通信配置 | 服务器地址、端口等 |
| 采集间隔 | 多久采集一次数据 |
| 告警阈值 | 液位/压力等告警值 |

#### 关键信号/引脚

| 信号名 | 方向 | 功能说明 |
|--------|------|----------|
| SCL | MCU→EEPROM | **时钟线**，同步通信 |
| SDA | 双向 | **数据线**，传输数据 |
| WP | 输入 | **写保护**，高电平时禁止写入 |

### 3.6 ADC采集模块 —— 板子的"感官"

#### 一句话总结
> **ADC（模数转换器）把连续的电压信号"翻译"成MCU能理解的数字**，就像人的感官把外界信息翻译成大脑能处理的神经信号。

#### 基本概念

**什么是ADC？**

- **模拟信号**：电压/电流是连续变化的（如水温可以是25.5°C、25.51°C...）
- **数字信号**：MCU只能处理0和1，离散的、不连续的

**ADC的工作**：把电压值（如2.53V）转换成数字（如 2067），MCU就能处理这个数字了。

#### 本板ADC通道配置

| 通道 | 信号名 | 测量内容 | 信号类型 | 说明 |
|------|--------|----------|----------|------|
| ADC_IN0 | MCU_ADC_BAT | 电池电压 | 电压型 | 直接测电压 |
| ADC_IN1 | MCU_AI_YL_I | 液位(电流) | 电流型 4-20mA | 两线制传感器 |
| ADC_IN2 | MCU_AI_LL | 液位(电压) | 电压型 0-10V | 三线制传感器 |
| ADC_IN3 | MCU_AI_YW | 液位(电压) | 电压型 0-10V | 备用通道 |
| ADC_IN4 | MCU_AI_YL_V | 流量反馈 | 电压型 0-10V | 流量计信号 |
| ADC_IN5 | MCU_AI_PRS0 | 压力反馈0 | 电流型 4-20mA | 工业压力变送器 |
| ADC_IN6 | MCU_AI_PRS1 | 压力反馈1 | 电压型 0-10V | 备用压力通道 |

#### 信号类型说明

| 类型 | 范围 | 说明 | 接法 |
|------|------|------|------|
| **电压型** | 0-10V 或 0-5V | 传感器输出电压信号 | 直接接ADC |
| **电流型** | 4-20mA | 工业标准，抗干扰强 | 串联采样电阻转电压 |

**为什么工业上用4-20mA而不是0-20mA？**
- 4mA表示"传感器正常工作"的最小值
- 0mA可以表示"传感器坏了/断线"
- 即使线路很长，电流信号衰减也小

### 3.7 蓝牙模块 —— 板子的"近距离遥控器"

#### 一句话总结
> **蓝牙模块提供近距离无线通信**，主要用于现场调试、设备配置，无需打开控制箱就能修改参数。

#### 功能特点

| 功能 | 说明 |
|------|------|
| **配置参数** | 通过手机APP修改设备IP、采集间隔等 |
| **固件升级** | 近距离无线更新程序（DFOTA） |
| **状态查看** | 用手机查看实时数据 |
| **调试诊断** | 连接电脑查看日志 |

#### 与MCU的连接

MCU和蓝牙模块通过**串口（UART）**通信，但因为电压不同，中间经过了**电平转换芯片（RS0108）**。

| 信号 | 说明 |
|------|------|
| BT_TXD | 蓝牙发送给MCU |
| BT_RXD | MCU发送给蓝牙 |
| BT_EN | 蓝牙使能（控制开关） |
| BT_STA | 蓝牙状态（连接/断开） |

### 3.8 GPS模块 —— 板子的"定位器"

#### 一句话总结
> **GPS模块获取设备所在位置的经纬度**，用于资产追踪、巡检打卡等场景。

#### 功能特点

| 功能 | 说明 |
|------|------|
| **定位** | 获取经纬度坐标 |
| **时间同步** | 获取准确的UTC时间 |
| **速度计算** | 计算移动速度 |

#### 与MCU的连接

MCU和GPS模块通过**串口（UART）**通信，波特率通常为9600或115200。

| 信号 | 说明 |
|------|------|
| GPS_RX | MCU发送给GPS |
| GPS_TX | GPS发送给MCU |
| GPS_ON | 休眠控制（低电平休眠） |
| GPS_NRST | 复位控制（低电平复位） |

### 3.9 RS232/RS485模块 —— 板子的"有线串口"

#### 一句话总结
> **RS232和RS485是工业现场最常用的有线通信接口**，用于连接PLC、变频器、采集器等外部设备。

#### RS232 vs RS485 对比

| 特性 | RS232 | RS485 |
|------|-------|-------|
| **传输距离** | 短（<15米） | 长（<1200米） |
| **通信方式** | 点对点 | 总线型（可接多个设备） |
| **抗干扰** | 较弱 | 强（差分信号） |
| **电平** | ±15V | ±5V |
| **设备数量** | 1对1 | 最多32个设备 |
| **本板应用** | 调试端口 | 连接工业设备 |

#### 本板接口定义

**RS232接口：**
| 信号 | 说明 |
|------|------|
| RS232_TXD | 发送数据 |
| RS232_RXD | 接收数据 |

**RS485接口：**
| 信号 | 说明 |
|------|------|
| RS485_TXD | 发送数据 |
| RS485_RXD | 接收数据 |
| RS485_TXE | 发送/接收方向控制 |

## 📚 第六章：术语表

### 核心术语解释

| 术语 | 英文全称 | 大白话解释 |
|------|----------|------------|
| **MCU** | Microcontroller Unit | 微控制器，简单说就是"微型电脑"，能独立工作 |
| **ADC** | Analog-to-Digital Converter | 模拟转数字转换器，把电压值变成数字 |
| **I2C** | Inter-Integrated Circuit | 两根线的串行通信协议，就像两人用对讲机对话 |
| **SPI** | Serial Peripheral Interface | 四根线的高速通信协议，比I2C快，但连线多 |
| **UART** | Universal Asynchronous Receiver/Transmitter | 串口，最简单的通信方式，就像发短信 |
| **GPIO** | General Purpose Input/Output | 通用输入输出，MCU的"万能接口" |
| **Flash** | Flash Memory | 闪存，断电不丢失的存储器 |
| **EEPROM** | Electrically Erasable Programmable ROM | 电可擦除可编程只读存储器，可反复修改的存储芯片 |
| **LTE** | Long Term Evolution | 4G网络的正式名称 |
| **Cat.1** | Category 1 | 4G的一个级别，中速够用、省电省钱 |
| **RS232** | Recommended Standard 232 | 串口通信标准，短距离点对点通信 |
| **RS485** | Recommended Standard 485 | 串口通信标准，长距离总线通信 |
| **SIM** | Subscriber Identity Module | SIM卡，设备的"身份证" |
| **GPS** | Global Positioning System | 全球定位系统 |
| **GNSS** | Global Navigation Satellite System | 全球导航卫星系统（GPS+北斗+伽利略等） |
| **MQTT** | Message Queuing Telemetry Transport | 物联网常用协议，像快递单一样传输数据 |
| **TCP/IP** | Transmission Control Protocol/Internet Protocol | 网络通信基础协议 |
| **AT指令** | AT Commands | 控制模块的命令，像给模块发微信 |
| **GPIO复用** | Pin Multiplexing | 一个引脚可以当好几个功能用 |
| **电平转换** | Level Shift | 让不同电压的芯片能一起工作 |

### 更多专业术语

| 术语 | 解释 |
|------|------|
| **上拉电阻** | 平时把电平拉高，有信号时拉低 |
| **下拉电阻** | 平时把电平拉低，有信号时拉高 |
| **去耦电容** | 稳定电源电压，过滤噪声 |
| **TVS二极管** | 防雷击、防静电保护 |
| **看门狗** | 检测系统是否死机，死了就重启 |
| **波特率** | 串口通信速度，越高传得越快 |
| **DMA** | Direct Memory Access，直接访问内存，CPU不用操心数据搬运 |
| **中断** | 突发事件通知机制 |
| **Bootloader** | 引导程序，开机先跑这段代码 |

## 💡 第七章：嵌入式开发入门提示

### 给产品经理的建议

作为产品经理，你不需要会写代码，但了解以下关键点对项目沟通很有帮助：

### 1. 硬件开发流程

```
需求定义 → 原理图设计 → PCB Layout → 打样 → 焊接 → 调试 → 认证 → 量产
    │            │            │         │      │      │       │
   你要参与     芯片选型    布局走线    工厂   工厂   研发   标准检测
```

### 2. 嵌入式软件开发流程

```
需求分析 → 详细设计 → 编码调试 → 单元测试 → 集成测试 → 固件发布
    │            │          │         │         │        │
   你要确认     架构设计   研发写代码  验证每个模块  整体测试  升级到设备
```

### 3. 关键时间节点

| 阶段 | 典型周期 | 你需要关注 |
|------|----------|------------|
| 原理图设计 | 1-2周 | 芯片选型、接口定义 |
| PCB设计 | 1-2周 | 层数选择、布局合理性 |
| 样板制作 | 1-2周 | 工厂交期 |
| 硬件调试 | 2-4周 | 功能验证 |
| 固件开发 | 4-8周 | 协议对接、业务逻辑 |
| 小批量试产 | 2-4周 | 生产测试 |
| 认证测试 | 4-8周 | CE/FCC/CCC等 |

### 4. 常见问题预警

| 问题类型 | 典型表现 | 解决周期 |
|----------|----------|----------|
| 通信不稳定 | 数据丢包、延迟大 | 2-4周 |
| 功耗超标 | 电池撑不了预期时间 | 1-2周 |
| 抗干扰差 | 设备附近有干扰就异常 | 2-4周 |
| 固件bug | 特定条件下死机 | 不确定 |

### 5. 和研发沟通的技巧

**应该问的：**
- "这个芯片的供货周期是多少？有没有替代方案？"
- "这个功能的技术风险点在哪里？"
- "测试用例覆盖了哪些场景？"
- "固件升级的机制是怎样的？"

**不需要问的：**
- "具体代码怎么写？"（这是研发的活）
- "这个电阻选多少欧姆？"（除非你是硬件工程师）
