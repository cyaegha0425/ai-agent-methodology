# 验证码处理规则

> 来源：百科3.21详解

## 规则配置

```markdown
### 验证码自动处理
- 触发条件：
  - 邮件主题包含"验证码"或"verification code"或"code"
  - 或发件人在白名单中
- 操作：
  1. 提取邮件正文中的数字验证码
  2. 追加到 ./temp/codes.md，格式：时间 | 来源 | 验证码
- 优先级：50
```

## 验证码存储格式

```markdown
# 验证码记录

| 时间 | 来源 | 验证码 | 用途 |
|------|------|--------|------|
| 2024-01-15 10:30:00 | noreply@example.com | 123456 | 登录验证 |
| 2024-01-15 14:22:00 | security@company.com | 789012 | 支付验证 |
```

## 提取规则

### 常见验证码格式

- 纯数字：6位或4位（银行、短信）
- 字母数字混合（大小写敏感）
- 带连字符：XXX-XXX-XXX
- 带空格：XXX XXX XXX

### 正则表达式示例

```python
# 6位纯数字验证码
r'\b(\d{6})\b'

# 包含连字符的验证码
r'([A-Z0-9]{3}-[A-Z0-9]{3}-[A-Z0-9]{3})'

# 带标签的验证码
r'验证码[：:]?\s*(\d+)'
```

## 白名单发件人

对于可信来源的验证码邮件，可以放宽提取条件：

```markdown
# 验证码白名单
noreply@google.com
no-reply@github.com
security@major-platform.com
```

## 注意事项

- 验证码有时效性（通常5-10分钟）
- 建议配合剪贴板或自动填充工具使用
- 同一验证码可能被多次发送，需要去重
