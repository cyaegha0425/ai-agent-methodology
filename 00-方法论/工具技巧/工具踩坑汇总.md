# 工具踩坑汇总

> 从 TOOLS.md 提取，按技术栈分类。遇到坑先查这里。

## Playwright

| 坑 | 原因 | 解决方案 |
|----|------|----------|
| timeout单位是毫秒 | Playwright API设计 | 写3000而不是3，设置默认超时：`page.set_default_timeout(CFG["TIMEOUT"] * 1000)` |
| JS click不触发React事件 | RSI前端用React自定义事件 | 使用Playwright原生`page.click()`而非JS click |
| CDP连接失败 | localhost可能解析问题 | 使用127.0.0.1而非localhost |
| ctx.pages[0]不可用 | Edge预存tab没有Playwright事件绑定 | 必须用`ctx.new_page()`创建新页面 |
| page.evaluate()参数限制 | Playwright API限制 | 只接受1个额外参数，多参数用dict传递 |
| JS正则转义问题 | Python字符串转义规则 | JS正则中的`$`和`\d`在Python中需写为`\\$`和`\\d` |

## tkinter GUI

| 坑 | 原因 | 解决方案 |
|----|------|----------|
| bg=""空字符串报错 | Tcl解析限制 | 必须使用实色作为背景色，如`bg="#A8B4D4"` |
| PhotoImage被GC回收 | Python垃圾回收机制 | `ImageTk.PhotoImage`必须保持self引用，如`self._bg_image = ImageTk.PhotoImage(...)` |
| 子线程操作GUI崩溃 | tkinter线程安全限制 | 禁止从子线程调用after/update，使用queue+主线程轮询方案 |
| padx/pady元组格式无效 | tkinter版本限制 | 不支持`padx=(2,8)`格式，分开写`padx=2`或使用place布局 |

## RSI GraphQL API

| 坑 | 原因 | 解决方案 |
|----|------|----------|
| 响应是批量数组格式 | RSI API设计 | 遍历响应数组查找对应operationName的结果 |
| AddCredit金额不匹配导致结账失败 | RSI业务规则 | amount必须与商品价格精确匹配，价格获取优先级：手动填写 > 拦截器提取 > DOM读取 |
| SKU不在HTML中 | RSI前端渲染逻辑 | SKU在GraphQL响应的`listing.resources`中，需用拦截器提取 |
| 不带keywords参数走SSR | RSI路由规则 | 商品列表页必须带`keywords=`参数才会走SPA发GraphQL请求 |

## 其他

| 坑 | 原因 | 解决方案 |
|----|------|----------|
| 编译缓存导致代码不生效 | .pyd文件优先级高于.py | 开发阶段删除所有`*.pyd`和`__pycache__`目录，仅发布时编译 |
| VPN导致脚本崩溃 | 系统级网络劫持 | 建议关闭VPN运行脚本 |

## 经验教训

- **串口通信**：不通时第一件事用万用表量每一根线的通断（包括GND），不要猜。虚焊/断线是最常见的根因，尤其是GND不共地——信号没有参考地，电压会飘，示波器波形也异常。（2026-05-06 Valve_Vision_Mie项目经验）

- **UI改动铁律**：涉及GUI界面改动时，必须先做UI预排列（计算控件尺寸、弹窗长宽、可见性），生成预览图发给用户确认后才能改代码，严禁直接改完再让用户看效果。所有UI改动必须用Pillow画预览图发给用户确认，确认后再改代码。（2026-05-06）
