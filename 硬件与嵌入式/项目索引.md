## 嵌入式项目三工具协作流程（铁律）

### 工具分工
| 角色 | 工具 | 干什么 |
|---|---|---|
| 方案设计+记忆+管理 | 我（阿音） | 搜方案、出设计文档、归档、提醒、知识库维护 |
| 代码编写+调试 | TRAE IDE SOLO | 在设备上写代码、跑、调试，本地直接执行 |
| 交付文档生成 | TRAE SOLO MTC | 生成PPT、Word、数据报告等成品文档 |

### 项目流程
1. **我（方案设计）**：搜现有方案 → 功能拆解 → 出设计文档 → push到GitHub
2. **TRAE IDE SOLO（写代码）**：pull设计文档 → AI根据文档生成代码 → 本地跑+调试 → push代码
3. **我（验收归档）**：pull代码+反馈 → 记录问题经验 → 更新知识库 → 整理素材文档push
4. **TRAE SOLO MTC（交付文档）**：读素材文档 → 生成产品说明书PPT/操作手册Word/测试报告

### 交付物定义
- **我 → TRAE IDE**：Markdown设计文档（硬件BOM、模块拆解、伪代码、UI草图、配置定义）
- **TRAE IDE → 我**：GitHub代码仓库 + 运行反馈（截图/口头）
- **我 → TRAE SOLO MTC**：Markdown素材包（项目概述、技术要点、输出格式要求）
- **TRAE SOLO MTC → 最终用户**：PPTX/DOCX成品文档

### 协作原则
- **所有交接走GitHub或Markdown文档**，不靠口头传递
- **每一环拿到文档就能独立干活**，不用来回问
- **先文档后代码**：设计文档确认后才进IDE
