# 文档命名方法论

> 慧飞ISO文档命名规范与实操指南
> 最后更新：2026-05-12

---

## 一、慧飞文档编号体系

### 1.1 4阶架构回顾

| 层级 | 文件类型 | 编号前缀 | 数量 |
|------|----------|----------|------|
| 1阶 | 手册/方针 | HF-MGMT-xxx | 5个 |
| 2阶 | 程序文件 | HF-PROC-xxx | 12个 |
| 3阶 | 作业指导书/SOP | HF-WI-xxx | 按部门分 |
| 4阶 | 表单/记录 | HF-FRM-xxx | 按部门分 |

另有补充分类：
| 分类 | 编号前缀 |
|------|----------|
| 技术文件 | HF-TECH-xxx |
| 说明书 | HF-UM-xxx |

### 1.2 前缀含义

| 前缀 | 全称 | 含义 |
|------|------|------|
| HF-MGMT | Management | 管理/方针类 |
| HF-PROC | Procedure | 程序文件类 |
| HF-WI | Work Instruction | 作业指导书类 |
| HF-FRM | Form/Record | 表单/记录类 |
| HF-TECH | Technical | 技术文件类 |
| HF-UM | User Manual | 用户手册类 |

### 1.3 编号规则

**核心原则：序号独立**

- 每个前缀系列的流水号独立
- HF-MGMT-001 的001和HF-PROC-001 的001是独立的
- 按创建顺序分配，不复用已作废编号

**流水号格式：** 3位数字（001, 002, 003...）

**版本号格式：** V1.0, V1.1, V2.0（主版本.次版本）

## 二、文档命名规范

### 2.1 标准命名格式

```
{前缀}[-{部门后缀}]-{流水号3位}-{文档名称} {版本号}.{扩展名}
```

**说明：**
- 无部门后缀：HF-MGMT-xxx, HF-PROC-xxx, HF-TECH-xxx, HF-UM-xxx
- 有部门后缀：HF-WI-xxx, HF-FRM-xxx（部门后缀紧跟前缀）

### 2.2 命名示例

**表单类（按部门）：**
```
HF-FRM-RD-001-产品立项申请书 V1.0.docx
HF-FRM-MFG-001-生产任务工单 V1.0.xlsx
HF-FRM-QA-001-来料检验报告 V1.0.docx
```

**SOP类（按部门）：**
```
HF-WI-MFG-001-产品组装SOP V1.0.docx
HF-WI-MFG-003-固件烧录作业指导书 V1.0.docx
```

**程序文件类：**
```
HF-PROC-001-文件控制程序 V1.0.docx
HF-PROC-007-产品设计与开发控制程序 V1.0.docx
```

**手册类：**
```
HF-MGMT-001-公司组织架构图 V1.0.pdf
HF-MGMT-003-质量手册 V1.0.docx
```

### 2.3 脱敏版命名

对外提供的文档需脱敏处理，命名时在版本号前加"脱敏版"：

```
HF-WI-MFG-001-产品组装SOP 通用脱敏版 V1.0.docx
HF-FRM-RD-005-产品需求规格书 通用脱敏版 V1.0.docx
```

### 2.4 归档版命名

归档版本在文件名前加"归档-"：

```
归档-HF-FRM-RD-005-产品需求规格书 V1.0.docx
归档-HF-PROC-007-产品设计与开发控制程序 V2.0.docx
```

## 五、新建文档实操流程

### 5.1 命名流程

```
Step 1: 确定文档类型
  ├── 手册/方针 → HF-MGMT-xxx
  ├── 程序文件 → HF-PROC-xxx
  ├── 作业指导书 → HF-WI-xxx
  ├── 表单/记录 → HF-FRM-xxx
  ├── 技术文件 → HF-TECH-xxx
  └── 说明书 → HF-UM-xxx

Step 2: 确定部门归属（表单/SOP需要）
  └── 按部门选择后缀

Step 3: 查询编号台账，确认下一个流水号
  └── 查询 ./知识库/ISO文档管理/编号台账.md

Step 4: 组合生成文件名
  └── {前缀}[-{部门}]-{流水号3位}-{文档名称} V1.0.{扩展名}

Step 5: 在编号台账登记
  └── 追加编号、文档名称、创建日期
```

### 5.2 版本升级规则

| 情况 | 操作 |
|------|------|
| 小幅修改（错别字、格式调整） | V1.0 → V1.1 |
| 内容较大调整（流程变、要求变） | V1.x → V2.0 |
| 重大重构 | Vx.x → V1.0（重置版本号） |

### 5.3 编号作废规则

- 编号一旦分配，不复用
- 作废编号在编号台账备注栏注明"已作废-原因"
- 归档时文件名加"归档-"前缀，原编号保留

## 六、快速参考

### 6.1 前缀速查

| 你要创建的是... | 用这个前缀 |
|-----------------|------------|
| 公司方针、质量手册 | HF-MGMT-xxx |
| 跨部门流程、程序 | HF-PROC-xxx |
| 怎么做某件事（SOP） | HF-WI-xxx |
| 做完留证据（表格、报告） | HF-FRM-xxx |
| 原理图、PCB、BOM | HF-TECH-xxx |
| 用户手册、运维手册 | HF-UM-xxx |

### 6.2 命名检查清单

- [ ] 编号格式正确（HF-XXX-xxx）
- [ ] 流水号3位数字
- [ ] 部门后缀正确（如适用）
- [ ] 文档名称清晰、无歧义
- [ ] 版本号格式正确（V1.0）
- [ ] 已在编号台账登记
- [ ] 扩展名与文件类型匹配
