> 基于慧飞实际业务制定，适用于IoT硬件产品公司

## 文件命名格式（完整版）

```
{前缀}[-{部门后缀}]-{流水号3位}-{日期YYYYMMDD}-{当天编号3位}-{文档名称} {版本号}.{扩展名}
```

**示例：**
```
HF-UM-001-20260511-001-铁丝双头阀-使用说明书 V1.0.docx
HF-SOP-MFG-001-20260512-001-产品组装SOP V1.0.docx
HF-FRM-RD-005-20260512-001-产品需求规格书 V1.0.docx
```

**模板命名（无日期/当天编号）：**
```
HF-UM-001-使用说明书模板-通用脱敏版 V1.0.docx
```

## 总体架构：4阶文件体系

| 层级 | 文件类型 | 说明 | 编号前缀 |
|------|----------|------|----------|
| 1阶 | 手册/方针 | 公司最高层级，阐述质量方针、目标、范围 | HF-MGMT-xxx |
| 2阶 | 程序文件 | 跨部门流程，规定"谁、在什么时间、做什么" | HF-PROC-xxx |
| 3阶 | 作业指导书/SOP | 具体"如何做"的详细操作说明 | HF-SOP-xxx |
| 4阶 | 表单/记录 | 做完事情后填写的证据（表格、报告、日志） | HF-FRM-xxx |

另有补充分类：
| 分类 | 说明 | 编号前缀 |
|------|------|----------|
| 技术文件 | 原理图/PCB/BOM/结构图等技术资料 | HF-TECH-xxx |
| 说明书 | 用户手册、运维手册 | HF-UM-xxx |

## SOP vs 表单/记录 区分

| 概念 | 一句话解释 | 例子 |
|------|------------|------|
| SOP | 教你怎么做 | 《智慧阀门组装作业指导书》 |
| 表单/记录 | 做完后留下证据 | 《出厂检测报告》 |

**实例：智慧阀门固件烧录**

**SOP：** HF-SOP-MFG-003-20260512-001 固件烧录作业指导书
- 内容：准备烧录器和线缆 → 连接阀门主板 → 打开烧录软件 → 选择固件文件 → 点击"烧录" → 观察绿色指示灯 → 贴上"已烧录"标签

**表单/记录：** HF-FRM-MFG-005-20260512-001 固件烧录记录表
- 内容：谁烧的、哪一批、烧录结果

**总结：** SOP告诉你怎么烧，表单记录你烧了没有、烧得对不对。
